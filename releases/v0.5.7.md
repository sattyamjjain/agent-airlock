## The single highest-leverage change: manifest-only mode

`mcp_spec/manifest_only_mode.py` ships the design that makes
runtime sanitization unnecessary because **argv never originates
from runtime input**:

```python
from agent_airlock import (
    ManifestRegistry,
    StdioManifest,
    launch_from_manifest,
)

registry = ManifestRegistry()
registry.register(
    StdioManifest(
        manifest_id="local-fs",
        command=("uvx", "mcp-server-everything"),
        env_allowlist=frozenset({"PATH", "HOME"}),
        signer="sre-team",
    ),
    signing_key=key,
)

# Anywhere in your runtime ŌĆö no argv flows from runtime input:
proc = launch_from_manifest("local-fs", registry, runtime_env=os.environ)
```

A runtime caller passing `command=["evil"]` raises
`ManifestRuntimeOverrideAttempted`. That's the load-bearing
guarantee.

`SecurityPolicy.stdio_mode` defaults to `"allowlist"` so the
v0.5.1 behaviour is preserved. Manifest-only mode is **opt-in for
hardening-conscious deployments**.

Latency: resolve+verify is **~3 ┬Ąs median**, register is **~20 ┬Ąs
median**. Locked into CI via ``tests/benchmarks/test_bench_manifest_mode.py``.

See [`docs/mcp/manifest-only-mode.md`](../docs/mcp/manifest-only-mode.md).

### CVE-2026-30615 ŌĆö Windsurf zero-click MCP-config auto-load

The only true zero-click in the OX-disclosure family ŌĆö attacker HTML
rewrites `.windsurf/mcp.json` from prompt injection alone, IDE
auto-launches the new server. Patched in Windsurf latest. The
class generalises to every IDE that auto-reads project-local MCP
config (VS Code, Cursor, Claude Code, JetBrains). New
`zero_click_config_guard.audit_config_diff` ships the
**diff-on-demand** API; the long-running daemon is queued for
v0.5.8. See
[`docs/cves/cve-2026-30615.md`](../docs/cves/cve-2026-30615.md).
